This article is the final entry in a series where I establish a non-prod cluster at home.

Let's recap quickly. In the first article, I wrote up a plan to build a new non-prod cluster. In the first milestone, I then got my virtualization layer set up and provisioned two clusters. In the second milestone, I established a means to connect to either cluster both from home or from the public internet.

The stage is now set to start building and adding capabilities to both clusters. Thankfully, I have non-prod to ensure things are working before sending new features to production.

Let's jump in.

## Spinning up prod-gen-2: re-running the scripts

In previous articles, I've mentioned that I've created a prod cluster, but I wanted to highlight here how I did it.

First, I ran `terraform apply` with my proxmox vm module. This was thankfully as simple as copy-paste, adjusting IP addresses and memory accordingly.

Next, once the VMs came up, I then ran my ansible playbook to bootstrap kubernetes onto the VMs. Yes, they came from templates, but the templated aspects are mostly just storage and vm-level concerns. Maybe one day, I'll turn the whole kubernetes node into a template.

Once kubernetes is up-and-running, then its time to bootstrap the cluster with flux. This is my favorite part, where all the hard work I've done starts to flow into the cluster and after many rounds of self-healing and dependencies resolving the cluster eventually tends toward a healthy state.
